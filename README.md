# Edge-ReStreamer

Edge-ReStreamer - это компактный и высокопроизводительный сервис для приема, парсинга и ре-трансляции видеопотоков с IP-камер (H.264) на edge-устройствах (x86/Raspberry Pi).

Проект разработан с упором на низкие задержки, zero-copy маршрутизацию кадров в памяти и глубокую аналитику. Сервис самостоятельно разбирает бинарную структуру видео (NALU, SPS/PPS), управляет таймстемпами и параллельно раздает поток в современные протоколы (LL-HLS и SRT), одновременно архивируя данные в S3-совместимое хранилище.

## Ключевые возможности

* **Глубокий парсинг H.264 (Demuxing):** Извлечение сырых NALU, определение границ GOP, чтение PTS/DTS таймстемпов на лету без полного транскодирования.
* **Low-Latency HLS (LL-HLS):** Генерация микро-чанков (Parts) в формате fMP4 и публикация тегов `#EXT-X-PART` для снижения задержки воспроизведения в браузере до 1-2 секунд.
* **SRT (Secure Reliable Transport):** Параллельная отдача MPEG-TS потока через протокол SRT (чистый Go-порт, без зависимостей от CGO) для сверхбыстрой (задержка <150 мс) P2P-передачи.
* **Zero-Copy Архитектура:** Раздача слайсов памяти с кадрами из демуксера в муксеры (HLS и SRT) через неблокирующие каналы Go.
* **S3 Cloud Backup:** Асинхронная выгрузка записанных HLS-сегментов в MinIO/AWS S3 без блокировки основного потока трансляции.
* **Observability (Мониторинг):**
    * **Prometheus Exporter:** Эндпоинт `/metrics` для интеграции с Grafana.
    * **Live Dashboard:** Встроенный веб-сервер с WebSocket (`/ws`) для отображения FPS, битрейта, потери кадров и загрузки CPU в реальном времени.
* **Graceful Shutdown:** Корректное завершение работы (дописывание сегментов, синхронизация с облаком) при получении `SIGTERM`.

## Технологический стек

* **Язык:** Go 1.26
* **Стриминг:** `bluenviron/mediacommon` (fMP4, MPEG-TS), `datarhei/gosrt` (SRT), `asticode/go-astits` (TS Demuxer)
* **Продюсер:** FFmpeg (интеграция через `exec.Command`)
* **Инфраструктура:** MinIO Go SDK (S3), Prometheus Client, Gorilla WebSocket

## Установка и запуск

### 1. Требования
* Установленный **Go 1.26**
* Установленный **FFmpeg 6.0+** (должен быть доступен в `PATH`)
* Доступ к любой RTSP-камере или тестовому потоку
* (Опционально) Запущенный сервер MinIO для тестирования загрузки в S3

### 2. Конфигурация (`config.yaml`)
Создайте файл `config.yaml` в корне проекта:

```yaml
camera:
  url: "rtsp://user:pass@10.0.0.42/stream1"
  timeout: "10s"
restream:
  hls:
    segment_duration: 2s
    playlist_entries: 6
    target_path: "./www/hls"
  srt:
    host: "127.0.0.1:9000"
    latency: 120ms
storage:
  s3_endpoint: "127.0.0.1:9000"
  bucket: "hls-backup"
  access_key: "minioadmin"
  secret_key: "minioadmin"
metrics:
  ws_port: 8080
  report_interval: 1s
```

### 4. Запуск

Перед запуском сервера вы можете поднять локального слушателя SRT (например, через VLC или FFplay) в отдельном терминале:
```bash
ffplay -fflags nobuffer -i "srt://0.0.0.0:9000?mode=listener"
```

Запустите ReStreamer:
```bash
go run cmd/main.go
```

## Эндпоинты

Приложение автоматически поднимает HTTP-сервер на порту `8080`:

* `http://localhost:8080/` — Прямая трансляция (HLS) и визуальный Dashboard в реальном времени.
* `ws://localhost:8080/ws` — WebSocket для получения Live-статистики (JSON).
* `http://localhost:8080/metrics` — Метрики в формате Prometheus для систем мониторинга.

## Дальнейшее развитие

* Интеграция `sync.Pool` на уровне демуксера для 100% исключения работы Garbage Collector'а при высоких битрейтах.
* Замена FFmpeg на чистую Go-имплементацию RTSP/RTP-клиента (через `gortsplib`) для сборки в единый бинарник.
* Поддержка динамической смены разрешения через встроенные фильтры.
* Распознавание цветовых пространств (BT.2020) из SPS NALU.
